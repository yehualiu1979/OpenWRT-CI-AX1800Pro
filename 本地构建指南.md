## 前置知识理解

### 什么是 OpenWRT？

OpenWRT 是一个基于 Linux 的嵌入式操作系统，主要用于路由器设备。它提供了高度可定制的环境，允许用户安装各种软件包，实现路由器的扩展功能。

**理解**：OpenWRT 本质上是一个精简的 Linux 发行版，专门为嵌入式设备设计。它使用 Buildroot 工具链来构建完整的系统，包括内核、用户空间工具、软件包等。

### 编译流程概述

```
源码 → 配置 → 下载依赖 → 编译工具链 → 编译内核 → 编译软件包 → 打包固件
```

**理解**：OpenWRT 编译是一个复杂的过程，需要：
1. 首先编译交叉编译工具链（在 PC 上编译能在路由器上运行的代码）
2. 然后编译 Linux 内核
3. 编译选定的软件包
4. 最后打包成可刷写的固件文件

### 为什么需要国内优化？

**理解**：
1. **网络访问**：GitHub、Google 等服务在国内访问缓慢或不稳定
2. **软件源**：默认的 apt 源在国内速度慢
3. **依赖下载**：编译过程中需要下载大量依赖包，国内网络环境下载困难
4. **镜像加速**：使用国内镜像可以显著提高下载速度

## 系统准备

### 硬件要求

| 资源 | 最低要求 | 推荐配置 | 说明 |
|------|---------|---------|------|
| CPU | 2核 | 4核+ | 编译是 CPU 密集型任务 |
| 内存 | 4GB | 8GB+ | 内存不足会使用交换空间，影响性能 |
| 磁盘 | 30GB | 50GB+ | 编译过程会产生大量临时文件 |
| 网络 | 稳定 | 代理+稳定 | 需要访问 GitHub 和软件源 |

**理解**：编译 OpenWRT 需要大量资源，特别是：
- CPU：需要同时编译多个软件包，多核可以并行编译
- 内存：编译过程中会加载大量代码到内存，内存不足会导致使用 swap，严重影响性能
- 磁盘：编译过程会产生约 20-30GB 的临时文件
- 网络：需要下载源码和依赖，国内网络环境需要优化

### 操作系统选择

**推荐**：Ubuntu 20.04 LTS 或 22.04 LTS

**原因**：
1. 稳定性好，长期支持
2. 软件包丰富
3. 社区支持好
4. 兼容性好

**不推荐**：
- Debian：软件包更新较慢
- CentOS/RHEL：包管理不同，可能遇到兼容性问题
- WSL：虽然可以运行，但性能可能不如原生 Linux

## 环境初始化

### 为什么需要 init_build_environment.sh？

**理解**：OpenWRT 编译需要大量的工具和依赖，手动安装非常繁琐且容易遗漏。`init_build_environment.sh` 脚本自动完成了以下工作：

1. **系统检测**：确保系统满足编译要求
2. **源配置**：根据网络环境配置最优软件源
3. **工具安装**：安装完整的编译工具链
4. **镜像配置**：配置国内镜像加速

### 执行初始化脚本

```bash
sudo bash -c 'bash <(curl -sL https://build-scripts.immortalwrt.org/init_build_environment.sh)'
```

**脚本做了什么？**

#### 1. 系统检查

```bash
# 检测系统版本
# Ubuntu 18.04 (bionic)
# Ubuntu 20.04 (focal)
# Ubuntu 22.04 (jammy)
# Ubuntu 24.04 (noble)
# Debian 10 (buster)
# Debian 11 (bullseye)
# Debian 12 (bookworm)
# Debian 13 (trixie)
```

**理解**：不同系统版本需要不同版本的编译工具：
- GCC 版本：9-13
- LLVM 版本：18
- Node.js 版本：18-22

#### 2. 网络检测

```bash
# 检测是否在中国网络
curl -s "myip.ipip.net" | grep -qo "中国" && CHN_NET=1

# 测试网络连通性
curl --connect-timeout 10 "baidu.com"
curl --connect-timeout 10 "google.com"
```

**理解**：
- 如果在中国网络，脚本会自动配置国内镜像源
- 如果能访问 Google，说明网络环境较好，可以使用官方源

#### 3. 软件源配置

**国内网络**：
```bash
# 替换为腾讯云镜像
deb https://mirrors.cloud.tencent.com/ubuntu/ focal main restricted universe multiverse
deb https://mirrors.cloud.tencent.com/ubuntu/ focal-security main restricted universe multiverse
deb https://mirrors.cloud.tencent.com/ubuntu/ focal-updates main restricted universe multiverse
```

**国外网络**：
```bash
# 使用官方源
deb http://archive.ubuntu.com/ubuntu/ focal main restricted universe multiverse
```

**理解**：
- 国内使用镜像源可以显著提高下载速度
- 不同地区可能需要使用不同的镜像源（清华、阿里云、腾讯云等）

#### 4. 添加额外软件源

```bash
# Node.js 源
deb https://deb.nodesource.com/node_22.x nodistro main

# Yarn 源
deb https://dl.yarnpkg.com/debian/ stable main

# GCC 工具链 PPA
deb https://ppa.launchpadcontent.net/ubuntu-toolchain-r/test/ubuntu focal main

# Git Core PPA
deb https://ppa.launchpadcontent.net/git-core/ppa/ubuntu focal main

# LLVM 工具链
deb https://apt.llvm.org/focal/ llvm-toolchain-focal-18 main

# Golang Backports
deb https://ppa.launchpadcontent.net/longsleep/golang-backports/ubuntu focal main

# GitHub CLI
deb [arch=amd64 signed-by=/etc/apt/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main
```

**理解**：
- OpenWRT 编译需要这些额外的工具
- PPA（Personal Package Archive）是 Ubuntu 的个人软件包存档
- 添加这些源可以获取最新版本的工具

#### 5. 安装依赖

```bash
apt install -y \
  build-essential ccache cmake ninja-build \
  gcc-10 g++-10 \
  clang-18 libclang-18-dev \
  nodejs yarn \
  golang-1.25-go \
  git subversion \
  python3 python3-pip \
  jq wget curl \
  # ... 更多依赖
```

**理解**：这些依赖可以分为几类：
- **编译工具**：GCC, Clang, CMake, Ninja
- **脚本语言**：Python, Node.js, Go
- **版本控制**：Git, SVN
- **压缩工具**：zstd, squashfs-tools
- **文档工具**：asciidoc, texinfo

#### 6. 配置国内镜像

```bash
# pip 镜像
pip3 config set global.index-url "https://mirrors.aliyun.com/pypi/simple/"

# npm 镜像
npm config set registry "https://registry.npmmirror.com" --global

# yarn 镜像
yarn config set registry "https://registry.npmmirror.com" --global

# go 镜像
go env -w GOPROXY=https://goproxy.cn,direct
```

**理解**：
- 编译过程中需要使用这些包管理器下载依赖
- 使用国内镜像可以大幅提高下载速度
- `goproxy.cn,direct` 表示优先使用国内镜像，如果失败则直接下载

#### 7. 安装 OpenWRT 特定工具

```bash
# UPX：可执行文件压缩工具
curl -fLO "https://github.com/upx/upx/releases/download/v5.0.2/upx-5.0.2-amd64_linux.tar.xz"
tar -Jxf "upx-5.0.2-amd64_linux.tar.xz"
cp -fp "upx-5.0.2-amd64_linux/upx" "/usr/bin/upx"

# padjffs2：JFFS2 文件系统填充工具
curl -fLO "https://raw.githubusercontent.com/openwrt/openwrt/main/tools/padjffs2/src/padjffs2.c"
gcc -Wall -Werror -o "padjffs2" "padjffs2.c"
cp -fp "padjffs2" "/usr/bin/padjffs2"

# po2lmo：LuCI 翻译文件转换工具
git clone --filter=blob:none --no-checkout "https://github.com/openwrt/luci.git" "po2lmo"
cd po2lmo
git config core.sparseCheckout true
echo "modules/luci-base/src" >> ".git/info/sparse-checkout"
git checkout
cd modules/luci-base/src
make po2lmo
cp -fp "po2lmo" "/usr/bin/po2lmo"
```

**理解**：
- **UPX**：压缩固件中的可执行文件，减小固件体积
- **padjffs2**：填充 JFFS2 文件系统的空白区域，提高性能
- **po2lmo**：将翻译文件（.po）转换为 LuCI 可用的格式（.lmo）

### 克隆源码

```bash
git clone --depth=1 --single-branch --branch $WRT_BRANCH $WRT_REPO ./wrt/
```

**参数说明**：
- `--depth=1`：浅克隆，只下载最新提交，减少下载量
- `--single-branch`：只克隆指定分支
- `--branch $WRT_BRANCH`：指定克隆的分支

**理解**：
- OpenWRT 源码仓库很大（数 GB），完整克隆需要很长时间
- 使用浅克隆可以显著减少下载时间
- 只需要最新的代码即可编译

## Feeds 更新

### 什么是 Feeds？

**理解**：Feeds 是 OpenWRT 的软件包管理系统，类似于 Linux 发行版的软件仓库。它包含了大量第三方软件包，可以方便地集成到固件中。

**Feeds 类型**：
1. **packages**：通用软件包
2. **luci**：LuCI Web 界面相关包
3. **routing**：路由相关包
4. **telephony**：电话相关包

### 更新 Feeds

```bash
./scripts/feeds update -a
./scripts/feeds install -a
```

**命令说明**：
- `feeds update -a`：更新所有 feeds
- `feeds install -a`：安装所有 feeds 中的包

**理解**：
- `feeds update` 会从 feeds 源下载最新的软件包信息
- `feeds install` 将 feeds 中的包链接到源码树中
- `-a` 参数表示处理所有 feeds

**执行过程**：
1. 从 feeds 源下载软件包索引
2. 下载或更新软件包
3. 将软件包链接到 `./package/feeds/` 目录
4. 更新 `./feeds/` 目录中的索引

#### 5. 更新软件包版本

```bash
UPDATE_VERSION() {
    local PKG_NAME=$1
    local PKG_MARK=${2:-false}
    local PKG_FILES=$(find ./ ../feeds/packages/ -maxdepth 3 -type f -wholename "*/$PKG_NAME/Makefile")
    
    # 获取最新版本
    local PKG_REPO=$(grep -Po "PKG_SOURCE_URL:=https://.*github.com/\K[^/]+/[^/]+(?=.*)" $PKG_FILE)
    local PKG_TAG=$(curl -sL "https://api.github.com/repos/$PKG_REPO/releases" | jq -r "map(select(.prerelease == $PKG_MARK)) | first | .tag_name")
    
    # 更新版本
    if [[ $NEW_VER =~ ^[0-9].* ]] && dpkg --compare-versions "$OLD_VER" lt "$NEW_VER"; then
        sed -i "s/PKG_VERSION:=.*/PKG_VERSION:=$NEW_VER/g" "$PKG_FILE"
        sed -i "s/PKG_HASH:=.*/PKG_HASH:=$NEW_HASH/g" "$PKG_FILE"
    fi
}

UPDATE_VERSION "sing-box"
UPDATE_VERSION "tailscale"
```

**理解**：
- 自动从 GitHub 获取软件包的最新版本
- 更新 Makefile 中的版本号和哈希值
- 可以检查预发布版本

### 执行 Handles.sh

```bash
/opt/wrt/Scripts/Handles.sh
```

**Handles.sh 做了什么？**

#### 1. 预置 HomeProxy 数据

```bash
if [ -d *"homeproxy"* ]; then
    HP_RULE="surge"
    HP_PATH="homeproxy/root/etc/homeproxy"
    
    # 清空旧规则
    rm -rf ./$HP_PATH/resources/*
    
    # 克隆 Surge 规则
    git clone -q --depth=1 --single-branch --branch "release" "https://github.com/Loyalsoldier/surge-rules.git" ./$HP_RULE/
    
    # 处理规则文件
    cd ./$HP_RULE/
    RES_VER=$(git log -1 --pretty=format:'%s' | grep -o "[0-9]*")
    echo $RES_VER | tee china_ip4.ver china_ip6.ver china_list.ver gfw_list.ver
    awk -F, '/^IP-CIDR,/{print $2 > "china_ip4.txt"} /^IP-CIDR6,/{print $2 > "china_ip6.txt"}' cncidr.txt
    sed 's/^\.//g' direct.txt > china_list.txt
    sed 's/^\.//g' gfw.txt > gfw_list.txt
    
    # 移动规则文件
    mv -f ./{china_*,gfw_list}.{ver,txt} ../$HP_PATH/resources/
    
    # 清理
    cd .. && rm -rf ./$HP_RULE/
fi
```

**理解**：
- HomeProxy 是一个代理工具，需要规则文件
- 从 Loyalsoldier/surge-rules 获取最新的规则
- 规则包括：
  - 中国 IP 列表
  - 直连列表
  - GFW 列表
- 将规则文件复制到 HomeProxy 的资源目录

#### 2. 修改 Argon 主题

```bash
if [ -d *"luci-theme-argon"* ]; then
    cd ./luci-theme-argon/
    
    # 修改字体
    sed -i "/font-weight:/ { /important/! { /\/\*/! s/:.*/: var(--font-weight);/ } }" $(find ./ -type f -iname "*.css")
    
    # 修改颜色
    sed -i "s/primary '.*'/primary '#31a1a1'/; s/'0.2'/'0.5'/; s/'none'/'bing'/; s/'600'/'normal'/" ./luci-app-argon-config/root/etc/config/argon

**理解**：
- 增加交换空间可以缓解内存不足
- 减少并行任务可以降低内存使用

sudo bash -c 'bash <(curl -sL https://build-scripts.immortalwrt.org/init_build_environment.sh)'

# 3. 克隆项目
cd /opt
git clone https://github.com/你的用户名/wrt.git
cd wrt

export WRT_CONFIG="IPQ60XX-WIFI-YES"

cd ./wrt/
export WRT_HASH=$(git log -1 --pretty=format:'%h')

# 6. 赋予脚本权限

# 7. 更新 Feeds

# 8. 自定义软件包
cd ./package/

/opt/wrt/Scripts/Settings.sh

# 10. 下载依赖
make download -j$(nproc)

# 11. 编译固件
make -j$(nproc)

# 12. 打包固件
mkdir ./upload/
cp -f ./.config ./upload/Config-"$WRT_CONFIG"-"$WRT_SOURCE"-"$WRT_DATE".txt

    NEW_FILE="$WRT_SOURCE"-"$NAME"-"$WRT_DATE"."$EXT"

find ./bin/targets/ -type f -exec mv -f {} ./upload/ \;
make clean -j$(nproc)
```

## 附录：项目配置说明

### 禁用国内镜像配置

**重要**：本项目配置了禁用 OpenWRT 源码中的国内镜像源。

#### 配置位置

在 `.github/workflows/WRT-CORE.yml` 文件中：

```yaml
- name: Clone Code(拉取编译的固件源码)
  run: |
    git clone --depth=1 --single-branch --branch $WRT_BRANCH $WRT_REPO ./wrt/

    cd ./wrt/ && echo "WRT_HASH=$(git log -1 --pretty=format:'%h')" >> $GITHUB_ENV

    # GitHub Action 移除国内下载源
    PROJECT_MIRRORS_FILE="./scripts/projectsmirrors.json"
    if [ -f "$PROJECT_MIRRORS_FILE" ]; then
      sed -i '/.cn\//d; /tencent/d; /aliyun/d' "$PROJECT_MIRRORS_FILE"
    fi
```

#### 配置说明

这个配置会在克隆源码后自动执行：

1. **查找配置文件**：检查 `./scripts/projectsmirrors.json` 是否存在
2. **删除国内镜像**：使用 `sed` 命令删除包含以下关键词的行：
   - `.cn/` - 中国域名
   - `tencent` - 腾讯镜像
   - `aliyun` - 阿里云镜像

#### 为什么需要禁用国内镜像？

**理解**：OpenWRT 源码中的 `projectsmirrors.json` 可能包含一些国内镜像源，但这些镜像可能：
- 不稳定或已失效
- 更新不及时
- 与官方源不同步
- 可能包含旧的或不完整的软件包

移除这些镜像源后，编译时会使用官方源，确保获取最新的代码和依赖。

#### 国内编译建议

如果在国内网络环境下编译，建议：

1. **使用代理访问官方源**
   ```bash
   export http_proxy=http://127.0.0.1:7890
   export https_proxy=http://127.0.0.1:7890
   
   git config --global http.proxy http://127.0.0.1:7890
   git config --global https.proxy http://127.0.0.1:7890
   ```

2. **不要禁用这个配置**
   - 避免使用过期的国内镜像
   - 确保使用最新的官方源

3. **配置系统镜像源**
   ```bash
   # apt 镜像（系统包管理）
   sudo sed -i 's/archive.ubuntu.com/mirrors.cloud.tencent.com/g' /etc/apt/sources.list
   
   # pip 镜像（Python 包）
   pip3 config set global.index-url "https://mirrors.aliyun.com/pypi/simple/"
   
   # npm 镜像（Node.js 包）
   npm config set registry "https://registry.npmmirror.com" --global
   
   # go 镜像（Go 包）
   go env -w GOPROXY=https://goproxy.cn,direct
   ```

4. **使用 GitHub 镜像（仅用于克隆）**
   ```bash
   # 使用 ghproxy 镜像克隆源码
   git clone https://mirror.ghproxy.com/https://github.com/VIKINGYFY/immortalwrt.git
   ```

#### 本地编译时的处理

如果使用本指南进行本地编译，建议：

```bash
# 克隆源码后，手动移除国内镜像
cd ./wrt/
PROJECT_MIRRORS_FILE="./scripts/projectsmirrors.json"
if [ -f "$PROJECT_MIRRORS_FILE" ]; then
    sed -i '/.cn\//d; /tencent/d; /aliyun/d' "$PROJECT_MIRRORS_FILE"
fi
```

**注意**：这个步骤已经包含在本指南的"源码获取"章节中，执行时会自动完成。

#### 总结

- **项目默认禁用 OpenWRT 源码中的国内镜像**
- **原因是确保使用最新的官方源**
- **国内编译时应使用代理访问官方源**
- **系统包管理器可以配置国内镜像加速**
- **不要禁用项目中的这个配置**
